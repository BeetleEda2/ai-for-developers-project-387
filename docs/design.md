# Design First: Сервис бронирования звонков

Упрощённый сервис бронирования звонков по мотивам Cal.com. Этот документ
фиксирует доменную модель, API-контракт и сценарии для шага **Design First**.

## Контекст

- В системе один заранее заданный владелец календаря (Owner).
- Регистрации и авторизации нет. Разделение ролей логическое.
- Слоты фиксированные: 30 минут.
- Рабочие часы владельца: 09:00–18:00 по таймзоне владельца (18 слотов/день).
- Окно бронирования: ближайшие 14 дней, прошедшее время исключается.
- Слот доступен, только если время свободно.
- Время старта бронирования уникально **глобально**: нельзя создать две записи
  на одно и то же время, даже если типы событий разные.

## Акторы

| Актор | Описание | Что делает |
|-------|----------|------------|
| Owner | Единственный владелец, предопределён, без логина | Создаёт типы событий; смотрит список предстоящих встреч |
| Guest | Без аккаунта | Смотрит типы событий и слоты; создаёт бронирование |

## Доменные сущности

| Сущность | Описание | Ключевые поля |
|----------|----------|---------------|
| Owner | Владелец календаря (предопределён) | `id`, `name`, `timezone` |
| EventType | Тип события/звонка, создаёт владелец | `id`, `title`, `description?`, `durationMinutes` (=30), `createdAt` |
| Slot | Вычисляемый 30-мин интервал доступности (не хранится) | `start`, `end`, `available` |
| Booking | Бронирование, создаёт гость | `id`, `eventTypeId`, `start`, `end`, `guestName`, `guestEmail`, `notes?`, `createdAt` |

**Инвариант:** множество значений `Booking.start` уникально глобально.

## API-контракт

### Owner

| Метод | Путь | Актор | Описание | Коды |
|-------|------|-------|----------|------|
| GET | `/owner` | Owner, Guest | Профиль владельца (имя, таймзона) | 200 |

### Event Types

| Метод | Путь | Актор | Описание | Коды |
|-------|------|-------|----------|------|
| GET | `/event-types` | Owner, Guest | Список типов событий | 200 |
| GET | `/event-types/{id}` | Owner, Guest | Один тип события | 200, 404 |
| POST | `/event-types` | Owner | Создать тип события | 201, 422 |
| GET | `/event-types/{id}/slots` | Guest | Свободные слоты в окне 14 дней | 200, 404 |

**Правило окна 14 дней:** слоты и бронирования допустимы только в диапазоне
`[сейчас; сейчас + 14 дней]`. Для `GET /slots` это окно по умолчанию; запрос с
`from`/`to` за его пределами обрезается. Для `POST /bookings` нарушение окна
(а также прошлое время, невыровненность по 30-минутной сетке, выход за рабочие
часы 09:00–18:00) даёт 422.

### Bookings

| Метод | Путь | Актор | Описание | Коды |
|-------|------|-------|----------|------|
| POST | `/bookings` | Guest | Создать бронирование | 201, 409, 422 |
| GET | `/bookings` | Owner | Список предстоящих встреч | 200 |
| GET | `/bookings/{id}` | Owner, Guest | Детали бронирования | 200, 404 |

Ключевые ответы об ошибках:

- **404 Not Found** — запрошенный ресурс (тип события, бронирование) не найден.
- **409 Conflict** — выбранное время уже занято другим бронированием.
- **422 Unprocessable Entity** — время вне окна 14 дней, не выровнено по
  30-минутной сетке, вне рабочих часов или в прошлом.

## Покрытие сценариев

| Сценарий | Эндпоинт |
|----------|----------|
| Владелец создаёт тип события | `POST /event-types` |
| Владелец смотрит предстоящие встречи | `GET /bookings` |
| Гость смотрит типы событий | `GET /event-types` |
| Гость смотрит доступные слоты | `GET /event-types/{id}/slots` |
| Гость создаёт бронирование | `POST /bookings` |
| Гость пытается занять занятое время | `POST /bookings` → 409 |
| Гость указывает невалидное время | `POST /bookings` → 422 |
| Гость/клиент получает данные владельца | `GET /owner` |

## Открытые вопросы (вне текущего шага)

- Отмена/перенос бронирования.
- Уведомления гостю и владельцу.
- Поддержка нескольких владельцев.
